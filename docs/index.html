🚀 Getting Started

Gitea Interact is a comprehensive Linux desktop application built with PyQt6 for managing your Git repositories and Gitea servers without using the command line.

Installation & Setup

  1. Clone or download the app from your Gitea server
  2. Install system dependencies: sudo apt install python3-pyqt6 python3-pip git
  3. Install Python dependencies: pip install -r requirements.txt
  4. Run the application: python3 main.py
💡 Quick Start: The welcome screen provides a comprehensive guide to all features. Right-click on repositories and files to access most Git operations!

📁 Repository Management

Adding Repositories

There are several ways to add repositories to Gitea Interact:

🔗 Single Repository

Gitea Menu → Add a Repo

Enter your repository URL (e.g., ssh://git@192.168.1.100:222/user/repo.git)

Note: Do not include "git clone" - just paste the URL

📥 Bulk Download

Gitea Menu → Bulk Download User Repos

Select a configured API server, enter a username, and download all their repositories at once

Repository Tree Features

  • 🔄 Refresh Button: Syncs the tree with external file system changes
  • 📊 Repository Status: Shows current commit information via right-click menu
  • 📁 File Navigation: Click to expand folders, double-click to open files
  • 🎯 Context Menus: Right-click for repository and file-specific operations

⚡ Git Operations

Repository-Level Operations (Right-click Repository)

📝 Add Changes

Stage all modified files for commit

💾 Commit Changes

Commit staged changes with a custom message

⬆️ Push to Server

Push committed changes to the remote repository

⬇️ Pull from Server

Pull and merge changes from the remote repository

📋 View Logs

Open the Git logs viewer in the right panel

ℹ️ Show Commit Info

Display detailed information about the current commit

Git Logs Viewer

The logs viewer provides comprehensive commit history management:

  • 📋 Commit List: Select any commit to view detailed information
  • 🔄 Safe Revert: Switch to a previous commit while preserving history (allows navigation back and forth)
  • 🗑️ Destructive Revert: Reset to a previous commit, permanently deleting subsequent commits
  • 📁 File-Specific Logs: Right-click files to view their individual commit history

Conflict Resolution

Gitea Interact handles various Git conflict scenarios:

  • Merge Conflicts: Automatic detection with resolution options (Resolve, Skip, Abort)
  • Divergent Branches: Options for merge, rebase, fast-forward, or overwrite local changes
  • Non-Fast-Forward Push: Force push, pull & merge, or pull & rebase options
  • Unmerged Files: Pre-operation checks with conflict resolution guidance

⚙️ Configuration

SSH Key Management

SSH Menu → Configure SSH Keys

  • 🔑 Create SSH Key: Generate new SSH key pairs (~/.ssh/id_rsa)
  • 👁️ View SSH Key: Display your public key with copy-to-clipboard functionality
  • 🔧 Test Connection: Verify SSH connectivity to your Git servers

API Configuration

Gitea Menu → Configure API

  • 🌐 Multiple Servers: Configure connections to multiple Gitea instances
  • 🔐 Token Management: Store and manage API tokens for each server
  • ✅ Connection Testing: Verify API connectivity and authentication
  • 📥 Bulk Operations: Download repositories from any configured server
SSH/API settings stored under ~/.config/Gitea-Interact/settings.json

Application Settings

Settings Menu

  • 🖥️ Fullscreen Mode: Toggle between windowed and fullscreen modes
  • 💾 Persistent Settings: Preferences are saved and restored on startup

🔧 Troubleshooting

Common Issues

🚫 Push Rejected (Non-Fast-Forward)

Problem: Server rejects push because local history has diverged

Solution: Use the conflict resolution dialog to choose between force push, pull & merge, or pull & rebase

⚠️ Merge Conflicts

Problem: Git operations result in merge conflicts

Solution: Use the built-in conflict resolution options (Resolve, Skip, Abort) with guided workflows

🔒 SSH Authentication Issues

Problem: Cannot connect to Git repositories via SSH

Solution: Use SSH Menu to create/view SSH keys and add the public key to your Gitea server

Git State Recovery

If you find yourself in an unexpected Git state:

  • Unmerged Files: Use the conflict resolution options in the context menus
  • Detached HEAD: This is normal after safe reverts - you can navigate between commits
  • Stuck Operations: Use "Abort" options in conflict dialogs to return to a clean state

Getting Help

  • Check the welcome screen for quick start guidance
  • Use right-click context menus for most operations
  • All destructive operations include confirmation dialogs
  • Error messages provide specific guidance for resolution
